What the IDS uses

Explanations Show Meaningful Network Evidence

How to read the figure

The bar ranking shows which features most influence the final IDS decision. It does not mean that a feature with value 0.04 is only slightly useful; it means its average contribution is large relative to other features inside that same explanation method.

Important: SHAP and SmoothIG use different units. Do not compare SHAP value 0.02 with SmoothIG value 6. Compare the feature order and interpretation.

Final ensemble explanation.Connection status, service concentration, login state, and error-rate features are logical IDS evidence.

Interpretation: the model is not using arbitrary features; it relies on security-relevant behavior such as successful connections, repeated services, host-level traffic, and error rates.

Feature importance values are method-specific; ranks and meanings are the defendable comparison4 / 10

Adversarial Stress Tests
PGD and mutable-feature PGD

Neural IDS Alone Remains Vulnerable

Full-feature PGD.A white-box stress test reaches 100% evasion on the original Torch Binary MLP, even at low eps.
Mutable-feature PGD after fine-tuning.More realistic attack: categorical and binary fields are frozen; reductions are strongest at eps 0.10 and 0.15.

Adversarial fine-tuning helps: mutable PGD evasion drops by 32.03 percentage points at eps 0.10 and 42.03 points at eps 0.15.

Limit: high-epsilon white-box PGD remains very strong, so the neural detector should not be deployed alone.

PGD attacks the differentiable neural model directly using gradients7 / 10
